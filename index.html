Three things I don't compromise on

  • Motion has to earn its place

    If an animation doesn't clarify state or hierarchy, it's cut — regardless of how good it looks in isolation.

  • Accessibility is a constraint, not a pass

    Keyboard support, contrast, and reduced-motion handling are built in from the first commit, not patched in at the end.

  • Every screen has a job

    Before I open a design tool, I can state the one thing a screen needs to help someone do. If I can't, it isn't ready.
